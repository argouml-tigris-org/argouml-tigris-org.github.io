Wiki: Making a release/Making a Stable release

For stable releases there are some extra things done to publish more information since the stable releases are used as reference for future work.

Do the following:

  1. Fix the javadoc version.
  2. Fix the ArgoUML documentation.
  3. Create Project Set Files that checks out Eclipse projects from the branch. (argoumlinstaller/psf/gen-psf-files.sh)
  4. Update the links to the documentation in the Using ArgoUML section throughtout the tigris site.
  5. Update the links to the documentation in the Users' Wiki.
  6. Upload on download sites:

    • http://www.heise.de - linus has the account.

In 0.32 we use the first beta and the start of the beta period to make a branch for the stable release. This means that we:

  • fix the javadoc version and
  • fix the ArgoUML documentation

at the first beta point.
