Code Generation Templates

Mapping UML models to Java source code without any additional information results in an simple element-to-codefragment which is in most cases not suitable for real projects. The MDA approach is a solution, since UML models can be enriched with guiding steretypes and tagged values. But then effort has to be put in the customization of the code generator. An easy-to-use GUI for defining how to process a list of steretypes and tagged values during code generation is proposed. These definitions can be stored and loaded as templates: A simple MDA support integration in ArgoUML.
